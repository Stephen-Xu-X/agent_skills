# 日记整理技能

## 触发条件

用户说以下任一内容时必须使用此技能：

- "整理今天的日记"
- "帮我写今天的日记"  
- "整理日记"
- "归档今天的记录"
- "帮我填日记"
- "整理一下"
- "帮我组织今天的日记"
- 或类似表达

不确定时优先触发而非忽略。

## 流程

### Step 1：读取日记文件

1. 根据对话上下文确定目标日期（今天 或 用户指定的日期）
2. 读取 `e:\Archive\obsidian\Personal Review\Diary\{日期}.md`
3. 读取 `e:\Archive\obsidian\Templates\Diary Template.md` 了解模板结构

### Step 2：分析原始内容

逐行/段分析用户的口语内容，归类到以下类别：

| 类别 | 说明 | 示例输入 → 输出 |
|------|------|----------------|
| **Todo（未完成）** | 计划要做、还没做的事 | "要给人家注册个Gemini" → `- [ ] 注册Gemini账号`，备注视情况 |
| **Todo（已完成）** | 今天已经做了的事 | "今天面了个试" → `- [x] 参加面试（公司名）` |
| **碎碎念** | 感想、吐槽、随机想法 | "热死了今天" → 保留原意 |
| **心情** | 情绪、感受 | "整个人都闷闷的" → Mood 子项 |
| **收获** | 今天学到的东西、成就 | "学会了用Templater" → 收获子项 |
| **明日方向** | 明天要做的事 | "明天还有个面试要面" → 明日方向子项 |

### Step 3：填充模板

按参考模板结构填入，保留用户原文中的所有细节。**注意**：

- ❌ **绝对不能无中生有**：用户没提到的内容留空，不补充
- ✅ **口语转书面**：把大白话转成简洁书面语，但保留关键信息
- ✅ **备注不丢失**：Todo 下的详细备注/子项必须保留，用缩进跟在 Todo 行下
- ✅ **多行备注**：如果备注有多条，每条独立一行缩进
- ❓ **内容太抽象**：主动问用户有没有细节要补充

### Step 4：输出草稿给用户确认

输出完整的日记草稿（含 YAML + 所有章节），问用户要不要修改或直接生成文件。

### Step 5：用户确认后生成文件

用户确认后，在工作目录下创建 `yyyy-mm-dd.md`，方便用户自行拖入 `Personal Review/Diary/` 文件夹。不锁定固定路径。

## 重要规则

1. **不脑补**：只整理用户说过的内容，不添加没提过的事
2. **不漏信息**：每个细节都要体现在输出中
3. **保持结构**：严格按照模板的章节和子模块呈现
4. **Todo 备注用缩进**：多行备注保持缩进关系
5. **口语转书面**：去口头禅、语气词，但保留原意
6. **存疑就问**：不确定的地方先问用户
7. **确认才写**：草稿给用户看，确认后再写入文件

## 周复盘模式

当用户说"整理本周复盘""整理周报""本周总结""这周怎么样"等指令时，进入周复盘模式。

### 流程

**Step 1：计算本周日期**

- 根据当前日期计算本周一 ~ 周日（YYYY-MM-DD）

**Step 2：读取每日日记**

- 循环 7 天，逐日读取日记文件 `日记/{日期}.md`
- 提取每篇日记的：mood、weather、Todo（[x]/[ ]含备注）、碎碎念、收获、明日方向

**Step 3：跨日合并 Todo**

- 同名任务跨日去重
- 同一任务只要任意一天被勾选 `[x]` → 算完成
- 从未勾选的 → 算未完成

**Step 4：AI 归纳生成报告**

按以下结构生成紧凑报告，规则：

- **详细但不照搬**：不逐日罗列数据，要按主题归纳合并
- **口语转书面**：保留原意但去口头禅
- **不无中生有**：用户没写的不要补
- **跨日关联**：找出跨日任务的变化
- **心情线**：描述每日心情及其影响因素
- **碎碎念主题**：把零碎想法按主题归类（如求职线、工具线）

### 输出格式

```markdown
---
created: {时间}
week: {ISO 周号}
period: {周一} - {周日}
mood_trend: {心情变化线}
total_done: N
total_pending: N
tags: [Review, Weekly]
---

# 第XX周 复盘（MM/DD - MM/DD）

> 一句话开篇

## 📊 本周完成情况

**完成 N 项，还有 N 项在进行中**

| 任务 | 状态 | 备注 |

## 📈 心情变化

逐日描述 + 整体总结

## 💬 碎碎念

按主题归纳

## 🧠 本周学到 / 搞定

## 🎯 下周方向
```

### 生成规则

- 工作日引用处添加 Obsidian 双链 `[[日记/YYYY-MM-DD]]`，便于跳转原始日记
- 任务备注中如有 `[[链接]]` 保留不动
- 在工作目录下创建 `{年份}-W{周号}-复盘.md`，方便用户拖入 `Personal Review/Diary/` 文件夹
