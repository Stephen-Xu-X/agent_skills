# 日记整理技能

## 触发条件

用户说以下任一内容时必须使用此技能：

- "整理今天的日记"
- "帮我写今天的日记"  
- "整理日记"
- "归档今天的记录"
- "帮我填日记"
- "整理一下"
- "帮我组织今天的日记"
- 或类似表达

不确定时优先触发而非忽略。

## 流程

### Step 1：读取日记文件

1. 根据对话上下文确定目标日期（今天 或 用户指定的日期）
2. 读取 `e:\Archive\obsidian\日记\{日期}.md`
3. 读取 `e:\Archive\obsidian\Templates\Diary Template.md` 了解模板结构

### Step 2：分析原始内容

逐行/段分析用户的口语内容，归类到以下类别：

| 类别 | 说明 | 示例输入 → 输出 |
|------|------|----------------|
| **Todo（未完成）** | 计划要做、还没做的事 | "要给人家注册个Gemini" → `- [ ] 注册Gemini账号`，备注视情况 |
| **Todo（已完成）** | 今天已经做了的事 | "今天面了个试" → `- [x] 参加面试（公司名）` |
| **碎碎念** | 感想、吐槽、随机想法 | "热死了今天" → 保留原意 |
| **心情** | 情绪、感受 | "整个人都闷闷的" → Mood 子项 |
| **收获** | 今天学到的东西、成就 | "学会了用Templater" → 收获子项 |
| **明日方向** | 明天要做的事 | "明天还有个面试要面" → 明日方向子项 |

### Step 3：填充模板

按参考模板结构填入，保留用户原文中的所有细节。**注意**：

- ❌ **绝对不能无中生有**：用户没提到的内容留空，不补充
- ✅ **口语转书面**：把大白话转成简洁书面语，但保留关键信息
- ✅ **备注不丢失**：Todo 下的详细备注/子项必须保留，用缩进跟在 Todo 行下
- ✅ **多行备注**：如果备注有多条，每条独立一行缩进
- ❓ **内容太抽象**：主动问用户有没有细节要补充

### Step 4：输出草稿给用户确认

输出完整的日记草稿（含 YAML + 所有章节），问用户要不要修改或直接生成文件。

### Step 5：用户确认后生成文件

用户确认后，在工作目录下创建 `yyyy-mm-dd.md`，方便用户自行拖入 `日记/` 文件夹。不锁定固定路径。

## 转换示例

### 示例 1

**用户原始输入：**
> 今天面了个试，感觉不咋地，回答也不行状态也不在，然后也没过了。还给人家注册了Gemini账号。明天还有个面试。

**整理后输出（仅填充的内容部分）：**
```
## 📋 今日待办
- [x] 参加面试
- [ ] 注册Gemini账号

## 💬 碎碎念
面试感觉不咋地，回答不行，状态也不在，最后没过

## 🌙 夜晚复盘

### 心情
- 面试没过，有点失落

### 今日收获
- 

### ⏩ 明日方向
- 还有一场面试
```

### 示例 2 — 带备注的 Todo

**用户原始输入：**
> 要给人家注册个Gemini账号，微信好友让我帮忙的。然后还要继续完善obsidian模板，特别是那个天气功能还没搞好。

**整理后输出：**
```
## 📋 今日待办
- [ ] 注册Gemini账号
      微信好友委托
- [ ] 继续完善obsidian模板
      天气功能还没搞好
```

## 重要规则

1. **不脑补**：只整理用户说过的内容，不添加没提过的事
2. **不漏信息**：每个细节都要体现在输出中
3. **保持结构**：严格按照模板的章节和子模块呈现
4. **Todo 备注用缩进**：多行备注保持缩进关系
5. **口语转书面**：去口头禅、语气词，但保留原意
6. **存疑就问**：不确定的地方先问用户
7. **确认才写**：草稿给用户看，确认后再写入文件
