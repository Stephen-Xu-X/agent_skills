# 最终归档规则

本技能支持围绕同一岗位进行多轮持续讨论。默认情况下，不在首次分析完成后立即生成本地文件。

仅当用户明确要求导出或归档时，才将当前岗位截至该时点的完整讨论内容整理为一份最终 Markdown 文件保存到 `~/Desktop/JD/`。

- 保存目录：`~/Desktop/JD/`
- 文件命名：`公司+岗位.md`
- 若公司或岗位缺失，分别使用 `未提供公司`、`未提供岗位`
- 导出内容不是第一次分析结果，而是结合本轮对话中所有补充讨论、修正、追问和总结后的最终版本
- 未收到明确导出指令前，不自动写入文件
- 保存前清洗文件名中的非法字符：`/ \ : * ? " < > |`
- 若同名文件已存在，可覆盖为最新版，或按用户要求追加日期/版本号
- 同一岗位的后续追问、补充判断、面试对练结论、简历修改建议，都应纳入最终归档内容
- 若用户在导出后继续补充同一岗位，可再次导出新版
- 文件名统一使用：`公司+岗位.md`
- 若需要保留历史版本，可追加 `v2`、日期或时间戳
