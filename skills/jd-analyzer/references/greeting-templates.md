# 问候语生成（Boss直聘跟进场景）

## 触发场景

用户在分析JD后表示"boss没回复""发问候语""打招呼""跟进""follow up""没回我"等意图时，**在完成五大模块分析后**，额外提供问候语优化服务。无需用户明确说"我要问候语"，只要对话中出现以上信号，就在分析末尾附上。

若用户**只粘贴了JD但没有说要分析**，且同时要求生成问候语，则直接进入问候语模块（跳过五大模块）。

## 问候语风格

Stephen Xu 的个人表达风格（基于其自我介绍书面/口语稿提炼）：
- **底层逻辑串联法**：习惯说"底层逻辑是相通的"，用同一逻辑线串联不同经历
- **链条式概括**：善用"对接需求→协调资源→推进节点→闭环交付"这类动作链条
- **从无序到有序**：强调自己擅长"把混乱的理清楚""从0到1搭建"
- **坦诚但不消极**：承认缺口但不自我贬低，用结构性归因解释离职/转行
- **节奏感**：短句为主，句式简洁，不爱堆形容词

问候语输出须贴合以上风格，不得过度包装，不得写成标准HR话术。

## 核心问候模板

在分析完JD后，基于JD要求个性化生成问候语，遵循以下三段式结构：

**① 破冰+自荐开场**（2-3句，用Stephen的底层逻辑串联法）
> 您好[称呼]，知悉投递人员较多且您工作繁忙，还是希望您能看到我这个信息，我想向您介绍下自己。
>
> 我虽然没有直接的[岗位]经验，但[前经历]的底层逻辑是相通的：[概括核心链路]。

**② 匹配点速览**（3个匹配点，用精简列表）
> 和这个岗位的匹配点：
> - **[JD关键词1]** [具体对应]
> - **[JD关键词2]** [具体对应]
> - **[有条理/流程思维]** [具体体现，如主导过SOP搭建/培训体系从0到1]

**③ 低门槛邀请**（1句）
> 如果合适我可以投递简历，期待进一步沟通。

所有问候语根据JD与候选人简历的匹配情况做个性化调整：
- 匹配度高 → 语气自信，用"底层逻辑是相通的"串联经历
- 匹配度中（有硬性缺口如学历/经验） → 语气诚恳，坦诚缺口+突出可迁移能力和学习能力
- 跨行/零经验 → 语气谦虚，用"从无序到有序"定位自身价值

## 用户自定义问候语优化

当用户自己写了一段问候语，要求"优化""改一下""按我语气改"时：

1. 分析用户原文的语气和用词风格（观察其常用动词、句式长短、是否用"底层逻辑""从0到1"等标志性表达）
2. 按以上三段式结构进行重组
3. **保持用户原始语气**，不做过度包装——不替用户说他们不会说的话
4. 输出2-3个版本供选择（推荐版+正式版+极简版）
5. 每个版本标注适用场景（如"这个版本更适合硬性缺口需坦诚的场景"）

## 实战示例（基于Stephen Xu真实风格）

以下为一个已实战验证的范例，符合Stephen的真实表达风格：

**场景**：Boss直聘投递采购岗，HR（龙经理）未回复，需要跟进

> 您好龙经理，知悉投递人员较多且您工作繁忙，还是希望您能看到我这个信息，我想向您介绍下自己。
>
> 商英专业，第一份工在工贸一体的建材工厂做 B2B 销售，兼搭 SOP、文件单证模板、新人培训框架这些内部项目；第二份在美企艾利丹尼森做项目部署，对接 lululemon 等品牌的 trail sample 开发，推进 O2D process — 具体就是对接 commercial 收集 requirement → 评估 spec & risk → 协调 material、PE 等部门 → 跟到 sample done。
>
> 坦白讲我没有直接的采购经验，但核心链路我是清楚的：需求确认 → sourcing → 供应商维护 → quotation 对比 → data import → order tracking 直至闭环。本质和我在外企做的是一样的，只是角色从对内协调换到了 procurement side。
>
> 几个匹配点：
> - **English** CET-6，美企全英文环境
> - **Excel** 熟练多函数联动和可视化，结合 AI agent 处理数据效率更高
> - **有条理** SOP 搭建、文件标准化、培训框架从0到1
>
> 如果合适我可以直接投简历，期待进一步沟通。

## 输出前检查清单

每次生成问候语后，对照检查：
- [ ] 是否过长（Boss直聘场景建议控制在200-300字，第一屏可见130-150字）
- [ ] 是否有 Stephen 的标志性表达（"坦白讲""底层逻辑""从0到1"等）
- [ ] 英文术语是否自然嵌入（sourcing、quotation、order tracking 等采购场景真实用词，不堆砌）
- [ ] 是否坦诚了关键缺口（如学历、经验不足）
- [ ] "有条理/系统化"有没有和具体场景挂钩（SOP、文件模板、培训框架等）
