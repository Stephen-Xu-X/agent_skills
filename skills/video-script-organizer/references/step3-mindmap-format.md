# Step 3: Topic Mindmap Format

Analyze the transcript's narrative structure. Divide into logically distinct topics with sub-topics. Each sub-topic must include rich, specific details from the transcript — enough that the `.md` file reads as a standalone reference.

```
# Mindmap: {input_filename}

- **[Main Topic 1]**
  中文翻译
  - **[Sub-topic A]**
    中文翻译
    - **[Key argument / example label in English]**
      Specific details from the transcript: context, examples, names, numbers, analogies — written as a mini-article. 2-5 sentences.
      中文翻译
    - **[Another key point]**
      More specific transcript details.
      中文翻译
    - **Source** / 出处
      "Verbatim quote from transcript"
      中文翻译
  - **[Sub-topic B]**
    中文翻译
    - **[Key point]**
      Detailed breakdown of this section's content.
      中文翻译
    - **Source** / 出处
      "Quote"
      中文翻译

- **[Main Topic 2]**
  中文翻译
  - **[Sub-topic C]**
    中文翻译
    - **[Key point]**
      Detailed content.
      中文翻译
    - **Source** / 出处
      "Quote"
      中文翻译
```

Constraints:
- Topics must correspond to content **explicitly discussed** in the transcript
- Specific details must be directly derived from the transcript (paraphrased or quoted)
- Each sub-topic should have at least one detailed content node beyond just Summary/Source
- Translations provide accessibility without altering or adding to the original meaning
- Use Markdown bullet hierarchy (`-`, `  -`, `    -`) for Markmap compatibility
